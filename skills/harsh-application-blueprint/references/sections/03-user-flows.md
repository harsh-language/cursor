# Section 3: User Flows

**Key:** `user_flows`

## Purpose

Defines how users move through the system to achieve goals.

## Questions

- What are the primary user goals?
- What steps are required to complete each goal?
- What alternate paths exist?
- Where can users abandon the flow?

## Required output

- End-to-end user journeys
- Flow diagrams (mermaid recommended)
- Happy paths
- Failure or abandon paths for each primary goal

## Minimum bar for `complete`

Assign **FL*** IDs. Every F* from §2 appears in at least one flow. Each primary goal: steps + happy + non-happy path.

## Compound Engineering skill

`ce-brainstorm` when flows are ambiguous, or in Phase 3 (review-only) for gap stress-test.
