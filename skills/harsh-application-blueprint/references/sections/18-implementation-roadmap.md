# Section 18: Implementation Roadmap

**Key:** `implementation_roadmap`

## Purpose

Defines build order and execution strategy.

## Questions

- What should be built first?
- What dependencies exist between phases?
- What milestones exist?
- What defines completion of each phase?

## Required output

- Development phases
- Milestones with deliverables
- Dependency map
- Delivery plan

## Minimum bar for `complete`

Phases are ordered with clear deliverables. Each phase has a "done when" statement. MVP path to a testable build is obvious.

## Note

After blueprint approval, `ce-plan` runs automatically for **Phase 1 / MVP only** (first phase listed here) unless `auto_plan: false`. Later phases become separate plans. This section captures product-facing phase intent, not file-level tasks.
