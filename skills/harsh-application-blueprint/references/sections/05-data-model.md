# Section 5: Data Model

**Key:** `data_model`

## Purpose

Defines all information managed by the system.

## Questions

- What entities exist?
- What attributes belong to each entity?
- How do entities relate to each other?
- What information must persist?

## Required output

- Entity definitions
- Attributes per entity
- Relationships
- Data schema (tables, documents, or equivalent)

## Minimum bar for `complete`

Every piece of user-visible data in flows has a home entity. Persistence requirements (local, cloud, session) stated.
