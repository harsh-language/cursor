# Section 17: Testing Strategy

**Key:** `testing_strategy`

## Purpose

Defines how quality is verified.

## Questions

- What must be tested?
- What platforms must be tested?
- What scenarios must be validated?
- What constitutes release readiness?

## Required output

- Test plan outline
- Test scenarios (especially for business rules and core flows)
- Acceptance criteria
- Release readiness definition

## Minimum bar for `complete`

Core flows and calculation/business rules have named test scenarios. Target platforms listed.
