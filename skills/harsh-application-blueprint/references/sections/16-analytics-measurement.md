# Section 16: Analytics & Measurement

**Key:** `analytics_measurement`

## Purpose

Defines how success is measured.

## Questions

- What events should be tracked?
- What metrics define success?
- What KPIs exist?
- What dashboards are required?

## Required output

- Event taxonomy (or "none for v1" with rationale)
- Tracking plan
- KPI definitions linked to §1 success criteria

## Minimum bar for `complete`

Every success criterion from Product Overview is either mapped to a metric/event or explicitly marked as qualitative with how it will be evaluated.
