# Section 14: Edge Cases & Failure States

**Key:** `edge_cases`

## Purpose

Defines behavior when things go wrong.

## Questions

- What happens when data is missing?
- What happens when requests fail?
- What happens when permissions are denied?
- What happens when users make mistakes?

## Required output

- Error states
- Empty states
- Recovery paths
- Exception handling approach

## Minimum bar for `complete`

Every primary flow from §3 has at least one documented failure or empty state. First-launch / zero-data experience defined.

## Compound Engineering skill

`ce-brainstorm` in Phase 3 (review-only) to find missing edge cases.
