# Section 2: Feature Requirements

**Key:** `feature_requirements`

## Purpose

Defines the capabilities of the product.

## Questions

- What can users do?
- What cannot users do?
- Which features are required for launch (MVP)?
- Which features are future scope?

## Required output

- Feature inventory
- MVP scope
- Future roadmap items
- Explicit non-goals

## Minimum bar for `complete`

Every MVP feature links to a user-facing capability. Assign **F*** IDs. Non-goals and future scope listed separately.

## Compound Engineering skill

`ce-ideate` when multiple directions or prioritization is needed. `ce-brainstorm` when requirements are incomplete.
