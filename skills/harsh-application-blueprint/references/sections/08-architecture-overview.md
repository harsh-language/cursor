# Section 8: Architecture Overview

**Key:** `architecture_overview`

## Purpose

Defines how system parts communicate.

## Questions

- What major systems exist?
- How do they interact?
- What services are internal?
- What services are external?

## Required output

- System diagram (mermaid or described)
- Data flow diagram
- Service relationships

## Minimum bar for `complete`

Major boundaries identified. Data flow from user action to storage and back is traceable for core flows.

## Note

High-level only here. Detailed HOW belongs in `ce-plan` after blueprint approval.
