# Section 15: Security & Privacy

**Key:** `security_privacy`

## Purpose

Defines protection of users and data.

## Questions

- What data is sensitive?
- Who can access what?
- How is authentication handled (if any)?
- How is data protected?

## Required output

- Authentication model (or "none" with rationale)
- Authorization model
- Data protection requirements

## Minimum bar for `complete`

Explicit for v1 scope. If no auth in MVP, state that and what data stays on-device vs would sync later.
