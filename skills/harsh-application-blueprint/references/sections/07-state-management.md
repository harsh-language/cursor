# Section 7: State Management

**Key:** `state_management`

## Purpose

Defines how application data is managed during runtime.

## Questions

- What data is global?
- What data is local (screen/session)?
- What data is temporary?
- What data must persist between sessions?

## Required output

- State architecture
- Ownership model (what owns what state)
- Persistence strategy

## Minimum bar for `complete`

Global vs local vs ephemeral is explicit for major entities. Sync or offline behavior referenced if applicable.
